Brigham and Women's Hospital Kiosk Application (Lead Software Engineer)

Role and Objective

Lead Software Engineer (Team of 11 SW Engs.)
Design and develop a functional kiosk application for Brigham and Women's Hospital, enhancing patient and visitor experience.

Link to GitHub Repository

Project Highlights:

Led a team of 13 developers in creating a user-friendly kiosk application for a touch billboard interface.
Implemented key features including:

  • Indoor Navigation: Assisted patients and visitors in navigating the complex hospital layout.
  • Text Message Directions: Provided convenient navigation assistance by sending directions directly to users' phones.
  • Hospital Communication: Offered a platform for accessing information and resources within the hospital.
  • Emergency Plans: Provided clear and accessible emergency protocols.
  • Technical Expertise: Wrote over 70,000 lines of Java and Python code across 11 software engineers..
  • Innovative Integration: Incorporated custom Amazon Alexa commands for hands-free interaction with the kiosk.
  • Solving Problems with Current Interfaces: I implemented code that sends directions to certain destination to your cell phone so you have a reference once leaving the kiosk
  • Successful Implementation: Presented the completed application to hospital staff and stakeholders, receiving positive feedback.

Technical Skills Developed:

  • Software Development Leadership
  • User Interface Design
  • Refactoring A* to deal with multi-level schematic based maps
  • Integration with Communication Tools
  • Project Management
  • Interpersonal Skills

Impact:

This project created a deployable application which streamlined hospital navigation and information access for patients and visitors, contributing to a more positive and efficient experience. It also taught us the valuable skill of how to work as an efficient amd effective team with so many engineers on one project.
